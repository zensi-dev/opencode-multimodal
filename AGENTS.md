# Project Guide

## Package Manager

- Use Bun for installs, scripts, and lockfile management.
- Do not add npm, pnpm, or yarn lockfiles. Root lockfile should be `bun.lock`.
- Run `bun install` after dependency changes. Prefer `bun install --frozen-lockfile` in verification/CI.

## Commands

Run these before considering work done:

- Build: `bun run build`
- Typecheck: `bun run typecheck`
- Tests: `bun run test`
- Lint/format: `bun run format`

## Architecture

- This package is an opencode plugin named `opencode-multimodal`.
- It has two plugin targets exposed through `package.json` exports:
  - `./server` -> `dist/server.js`, built from `src/server/index.ts`.
  - `./tui` -> `src/tui/index.tsx`, loaded by opencode TUI runtime as TSX.
- `src/shared/` holds reusable logic and types: config paths, config store, auth lookup, model metadata, provider package mapping, prompts, and utilities.
- `src/server/` registers backend hooks, detects unsupported attachments, chooses configured fallback models, calls Vercel AI SDK providers, caches results, and replaces attachments with text descriptions.
- `src/tui/` registers `/multimodal` and renders the configuration UI with OpenTUI/Solid components.
- `test/` uses Vitest for unit tests plus transform integration coverage.
- `dist/` is generated by `bun run build`; do not hand-edit generated build output unless explicitly requested.

## OpenCode Config

- `.opencode/opencode.json` and `.opencode/tui.json` self-load this package from `"plugin": [".."]` for local development.
- If `.opencode/*` config changes, restart opencode. Config is loaded at startup.
- `.opencode/package.json`, `.opencode/package-lock.json`, `.opencode/bun.lock`, and `.opencode/node_modules/` are local opencode dependency artifacts and are ignored.

## Notes

- `npm` in code/data often means npm package metadata or npm registry package name, not npm as the project package manager.
- Runtime provider packages are intentionally listed as dependencies so opencode can install and load plugin fallbacks from its Bun-managed cache.
